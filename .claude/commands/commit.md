# 변경사항 커밋

현재 모든 변경사항을 분석하여 커밋합니다.

## 추가 지시사항
$ARGUMENTS

## 작업 지침

1. `git status`와 `git diff`로 현재 변경사항을 확인합니다
2. `git log --oneline -5`로 기존 커밋 메시지 형식을 확인합니다
3. 변경사항을 분석하여 커밋 메시지를 작성합니다

## 커밋 메시지 형식

```
{type}: {주요 변경 요약}

- {태스크 1}
- {태스크 2}
- {태스크 3}
```

### Type 종류
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `refactor`: 리팩토링
- `style`: 스타일/포맷 변경
- `docs`: 문서 변경
- `chore`: 기타 변경

### 규칙
- 첫 줄은 기존 커밋 포맷(`{type}: {설명}`)을 따릅니다
- 본문은 `-`와 줄바꿈으로 태스크를 분리합니다
- 각 태스크는 구체적이고 명확하게 작성합니다
- 한글로 작성합니다

## 예시

변경사항: 타입 파일 분리, API import 경로 수정

```
refactor: 타입 시스템 모듈화

- common.ts: 공통 타입 및 상수 분리
- campaign.ts: 캠페인 관련 타입 정의
- filter.ts: 필터 상태 타입 추가
- API import 경로 업데이트
```

## 실행

1. 모든 변경 파일을 staging합니다
2. 위 형식에 맞춰 커밋을 생성합니다
3. 커밋 결과를 사용자에게 보여줍니다
