## **2. 데이터 명세 및 활용 가이드**

아래 제공되는 `db.json` 를 사용해주세요.

[db.json](attachment:6514778f-1cc4-4d47-884d-4c1abc2764f3:db.json)

### **2.1 데이터 스키마 (Data Schema)**

```tsx
// 1. Campaign: 캠페인 정보
interface Campaign {
  id: string; // 고유 식별자
  name: string; // 캠페인 명칭
  platform: "Google" | "Meta" | "Naver"; // 광고 매체
  status: "active" | "paused" | "ended"; // 상태 (진행중, 일시중지, 종료)
  budget: number; // 총 예산
  startDate: string; // 시작일 (YYYY-MM-DD)
  endDate: string | null; // 종료일 (YYYY-MM-DD)
}

// 2. DailyStat: 캠페인별 일간 성과 (1:N 관계)
interface DailyStat {
  id: string; // 고유 식별자
  campaignId: string; // 연관된 캠페인 ID
  date: string; // 성과 발생 날짜 (YYYY-MM-DD)
  impressions: number; // 노출수
  clicks: number; // 클릭수
  conversions: number; // 전환수
  cost: number; // 집행 비용
  conversionsValue: number | null; // 전환 가치 (매출액)
}
```

---

### **2.2 중요 제약 사항**

| **항목**             | 내용                                                                                         |
| -------------------- | -------------------------------------------------------------------------------------------- |
| **원본 유지**        | 제공된 `db.json` 파일은 수정 없이 원본 그대로 활용해야 합니다.                               |
| **비동기 통신 필수** | import 방식 금지하며, json-server, msw 등을 활용하여 API로 통신해야 합니다.                  |
| **API 예시**         | `GET /campaigns`, `GET/ daily_stats` (json-server 기반)                                      |
| **상태 영속성**      | 신규 등록 데이터는 브라우저 메모리 내에서만 유지되어도 무방합니다. (새로고침 시 초기화 허용) |

---

### 2.3 파생 지표 계산 및 전처리

파생 지표는 필터 조건에 해당하는 `dailyStats` 합산 데이터를 기준으로 실시간 계산합니다.

| 파생 지표    | 계산식                              |
| ------------ | ----------------------------------- |
| **CTR (%)**  | (총 클릭 수 / 총 노출 수) × 100     |
| **CPC (원)** | 총 집행 비용 / 총 클릭 수           |
| **ROAS (%)** | (총 전환 가치 / 총 집행 비용) × 100 |

<aside>
⚠️

**`평가 포인트`** 제공된 데이터에는 실무 환경에서 발생할 수 있는 **예외 상황**이 포함되어 있습니다.
**`db.json` 데이터를 그대로 사용하지 말고, 안전하게 처리한 뒤 사용해주세요.**

- **Division by Zero**: 분모가 0인 경우 (CTR, CPC 등 계산 시)
- **Null Safety**: 일부 필드가 null인 경우
- **데이터 정규화**: 필드 값의 포맷이 다른 경우
</aside>

## 3. 구현 요구사항(필수)

### 3.1 글로벌 필터

- 필터 조건 변경 시, 하단의 **모든 차트와 테이블 데이터가 실시간으로 동기화**되어야 합니다.
- 각 필터 조건은 **AND** 조합으로 적용됩니다.

| **필터**                                | **세부사항**                      | 제약사항                      |
| --------------------------------------- | --------------------------------- | ----------------------------- |
| **집행 기간**                           | 선택한 기간 내 집행 중인 캠페인과 |
| 해당 기간의 일별 데이터를 기준으로 집계 | 초기값(당월 1일~말일)             |
| **상태**                                | 진행중/일시중지/종료              | 초기값(전체) / 다중 선택 가능 |
| **매체**                                | Google / Meta / Naver             | 초기값(전체) / 다중 선택 가능 |
| **초기화**                              | 모든 조건을 초기값으로 복구       |                               |

---

### 3.2 일별 추이 차트

- 필터링된 캠페인의 일별 데이터를 시각화합니다.
- **차트 구성**: X축(날짜), Y축(수치), 범례(Legend) 필수
- **메트릭 토글**
  - **초기값**: '노출수', '클릭수' 모두 활성화 (중복 선택 가능)
  - **제약**: 최소 1개의 지표는 반드시 선택된 상태 유지
- **인터랙션**: 호버 시 해당 날짜의 수치를 툴팁으로 표시

> 차트 유형은 자유롭게 선택 가능합니다.

---

### 3.3 캠페인 관리 테이블

- **컬럼**: 캠페인명, 상태, 매체, 집행기간(시작일~종료일), 총 집행금액, CTR(%), CPC(원), ROAS (%)
- **정렬**: 집행기간, 총 집행금액, CTR, CPC, ROAS 컬럼에 대해 오름차순/내림차순 정렬
- **검색**: 캠페인명 실시간 검색 (**단, 테이블에만 적용**)
  - 검색 결과 건수 / 전체 건수 표시
- **페이지네이션**: 1페이지당 10건 노출
- **`평가 포인트`일괄 상태 변경**: 체크박스로 캠페인 선택 → 드롭다운으로 상태(진행중/일시중지/종료) 일괄 변경

---

### 3.4 캠페인 등록 모달

- **인터랙션:** [캠페인 등록] 버튼 클릭 시 모달 노출
- **유효성 검사:** 검사 실패 시, 해당 필드 하단에 에러 메시지 표시
- **`평가 포인트`상태 동기화:** **등록 성공 시 별도의 새로고침 없이** 목록과 대시보드 차트에 즉시 반영되어야 함
  - 신규 캠페인은 `daily_stats`가 없으므로, 테이블에서 지표는 0 또는 -으로 표시되어도 무방합니다.
- **자동 설정 값:**
  - 상태: `active (진행 중)`로 고정
  - 캠페인 ID: 고유한 식별값 자동 생성
- **데이터 영속성:** 브라우저 세션 내 유지 (새로고침 시 초기화 허용)

| **입력 필드** | **타입** | **필수** | **유효성 검사 규칙**                    |
| ------------- | -------- | -------- | --------------------------------------- |
| **캠페인명**  | 텍스트   | O        | 2자 ~ 100자                             |
| **광고 매체** | 선택     | O        | Google, Meta, Naver 중 택 1             |
| **예산**      | 숫자     | O        | 정수, 100원 ~ 10억 원                   |
| **집행 금액** | 숫자     | O        | 정수, 0원 ~ 10억 원, **예산 초과 불가** |
| **시작일**    | 날짜     | O        |                                         |
| **종료일**    | 날짜     | O        | 시작일 이후                             |

> 집행 금액은 실제 서비스에서는 일별 성과의 합산이나, 과제 편의상 초기값으로 직접 입력받습니다.

## 4. 구현 요구사항 (선택, 구현 시 가산점 반영)

<aside>
📌

필수 구현 사항은 아니나, 구현 여부와 완성도에 따라 최종 평가에 **가산점**이 부여됩니다.

</aside>

### 4.1 플랫폼별 성과 차트 (Donut)

- **메트릭 토글**: 비용 / 노출수 / 클릭수 / 전환수 (기본값: 비용)
- **차트**: 플랫폼별(Google / Meta / Naver) 비중 표시
- 메트릭 수치와 비중(%) 동시 표기
- **`평가 포인트`** 차트 클릭 시, 상단 글로벌 필터와 양방향 연동 (필터 적용/해제)
  - ex) 도넛의 Google 영역 클릭 → 상단 매체 필터에 'Google' 적용 → 모든 데이터 연동

---

### 4.2 캠페인 랭킹 Top3 (Bar)

- **메트릭 토글**: ROAS / CTR / CPC 기준 전환 가능 (기본값: ROAS)
- 선택한 메트릭 토글에 해당되는 상위 3개 캠페인 노출
- **`평가 포인트`정렬**: ROAS/CTR은 높을수록 ↑, CPC는 낮을수록 ↓ 상위 랭킹
